Memory Model

Four Types of Personal Memory

Every piece of knowledge is classified, embedded (384-dim), and stored in PostgreSQL with pgvector. Memories strengthen on access and decay when unused.

Architecture

Module Structure

Hexagonal architecture — port interfaces in core-domain, adapters in core-memory, Spring Boot app in core-api.

core-domain
No Spring dependency

Domain Types + Port Interfaces

PersonalMemory, MemoryType, CognitiveSession, PersonalContext records. PersonalMemoryStore and PersonalContextProvider port interfaces.

core-memory
Spring JDBC adapter

pgvector Store + Embedding

PGVectorPersonalMemoryStore: cosine similarity search, upsert. PersonalEmbeddingService: Ollama all-MiniLM-L6-v2 (384-dim), graceful fallback.

core-api
Port 8082

Spring Boot Application

PersonalContextController (Grid integration endpoint), PersonalMemoryController, Flyway migrations, actuator probes, CoreApiConfig wiring.

core-infra
Infrastructure

Docker + Migrations

Docker Compose (postgres-core on 5433, aether-core on 8082). Standalone Flyway migrations. Kubernetes Helm chart planned in Phase 6.

API Contract

REST Endpoints

The key endpoint is consumed by Aether Grid's AetherCoreBridgeAgent. The contract must not break without coordinated Grid updates.

MethodPathDescription
GET /api/v1/personal-context/{tenantId}/{userId} Personal context snapshot — consumed by Aether Grid
POST /api/v1/users/{userId}/memories Store a new personal memory (type + content)
GET /api/v1/users/{userId}/memories/count Total memory count for a user
DELETE /api/v1/users/{userId}/memories/{memoryId} Delete a specific memory
GET /actuator/health Liveness + readiness probes
